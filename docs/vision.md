# Vision

UX Methods is a User Experience Architecture and Design Knowledge Platform.

**Product Vision:** A multi-surface platform for accessing actionable user experience architecture and design knowledge.

- a pleasant and easy-to-use web interface for the discovery of UX methods and resources
- an approachable and welcoming platform for contributing knowledge to a mission-driven and community focused project
- a set of interoperable interfaces (or "surfaces," see below) for integrating product knowledge into diverse planning and decision-making workflows

**Project Vision:** An approachable and open set of pursuits and practices for learning about the intersection of composable content, symbolic knowledge modeling, and generative AI.

## Surfaces

1. **Website UI:** explore, learn, link, bookmark, share
2. **Website LD:** interoperability, discovery, sharing
3. **Fuseki:** explore, experiment
4. **Protégé** (GitHub data): explore, experiment, prototype, learn (Web Protégé?)
5. **[WebVOWL](https://service.tib.eu/webvowl/#iri=https://raw.githubusercontent.com/andybywire/ux-methods/refs/heads/main/graph/ontologies/uxmethods-core.ttl)** (GitHub data): discover, explain, demonstrate
6. **Repository** (GitHub data): Claude Code optimized project collaborator orientation, exploration, contribution
7. **Ontology Documentation** (subdomain): non-technical overview of the (eventual) ontology, how and where to use (e.g. [IPTC](https://sportschema.org), [BBC](https://www.bbc.co.uk/ontologies/bbc/#classes-ContentBrand))
8. **MCP:** a knowledge graph enabled access point for UXM knowledge
9. **CMS:** Sanity based authoring and content ops interface

## Characteristics

- freely links out to other resources
- focuses on the connections between methods, as opposed to being exhaustive
- open and experimental: every design decision and approach represents a learning opportunity, sometimes for how to be effective, sometimes for what not to do
